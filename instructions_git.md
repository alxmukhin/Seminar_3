# **Инструкция по использованию системы контроля версий Git**

## Что такое Git?

Git это программа предназначенная для управления версиями проекта и отслеживания изменений в нем.

## Инициализация репозитория

Чтобы инициализировать (создать) новый репозиторий необходимо использовать команду:
    
    git init

## Проверка состояния репозитория

Чтобы проверить состояния репозитория необходимо использовать команду:

    git status

## Добавление изменений в индекс

Чтобы добавить изменения в индекс для последующего коммита необходимо использовать команду:

    git add <file name>

## Добавление коммита

Для добавления коммита описывающего изменения файла необходимо использовать команду:

    git commit -m "commit description"

## Добавление коммита для всех изменений

Для добавления коммита описывающего все изменения отслеживаемого файла необходимо использовать команду:

    git commit -a -m "commit description"

## Просмотр истории коммитов

Для просмотра истории коммитов необходимо использовать команду:

    git log

## Просмотр истории коммитов построчно

Для вывода истории коммитов построчно необходимо использовать команду:

    git log --oneline

## Просмотр истории коммитов для всех веток

Для вывода истории коммитов по всем веткам необходимо использовать команду:

    git log --all

## Переход к определенному коммиту

Для перехода к определенному коммиту необходимо использовать команду:

    git checkout <commit hash>

## Переход к последнему изменению

Для перехода к последнему изменению необходимо использовать команду:

    git checkout master

## Просмотр различий между текущей версией и определенным коммитом

Для проверки различий между текущей версией и определенным коммитом необходимо использовать команду:

    git diff <commit hash>
